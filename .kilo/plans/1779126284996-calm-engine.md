# Plan: Email Signup Form → Mailchimp with Reward Link

## Summary

Add an email signup modal that appears when the user clicks "Download" on a freebie card. The form collects the user's email, submits it to Mailchimp via a TanStack Start server function (running on Cloudflare Workers), and passes a merge field identifying which freebie the user requested. Mailchimp automation then sends the reward email with the correct download link.

---

## Architecture

```
[FreebieDownloadCard] → click "Download"
        ↓
[SignupModal] (email input + submit button)
        ↓
[Server Function: subscribeWithReward]
        ↓
Mailchimp API (add member + REWARD merge field)
        ↓
Mailchimp Automation (sends email with *|REWARD_URL|* link)
```

---

## Implementation Steps

### 1. Create freebie config (`src/config/freebies.ts`)

A typed config array mapping each freebie to its metadata:

```ts
export type Freebie = {
  id: string           // e.g. "cartwheel-colors"
  title: string
  imageSrc: string
  rewardUrl: string    // the actual download link Mailchimp will include
}

export const freebies: Freebie[] = [
  {
    id: 'cartwheel-colors',
    title: 'Cartwheel Colors',
    imageSrc: '/images/cartwheel-colors.png',
    rewardUrl: 'https://your-domain.com/downloads/cartwheel-colors.pdf',
  },
  // ... add more as needed
]
```

### 2. Create the server function (`src/server/subscribe.ts`)

- Uses `createServerFn` from `@tanstack/react-start`.
- Uses `import { env } from 'cloudflare:workers'` to access Mailchimp credentials.
- Accepts `{ email: string, rewardUrl: string }` via `.inputValidator()`.
- POSTs to Mailchimp's `POST /lists/{audience_id}/members` endpoint with:
  - `email_address`
  - `status: "subscribed"` (single opt-in)
  - `merge_fields: { REWARD_URL: rewardUrl }`
- Returns `{ success: true }` or throws with a user-friendly error.
- Handles "Member Exists" error (Mailchimp 400) gracefully — updates the existing member's merge field via PATCH instead.

```ts
import { createServerFn } from '@tanstack/react-start'
import { env } from 'cloudflare:workers'

export const subscribeWithReward = createServerFn({ method: 'POST' })
  .inputValidator((data: { email: string; rewardUrl: string }) => {
    if (!data.email) throw new Error('Email is required')
    if (!data.rewardUrl) throw new Error('Reward is required')
    return { email: data.email.trim().toLowerCase(), rewardUrl: data.rewardUrl }
  })
  .handler(async ({ data }) => {
    const { MAILCHIMP_API_KEY, MAILCHIMP_AUDIENCE_ID, MAILCHIMP_DC } = env
    // ... Mailchimp API call
  })
```

**Cloudflare Workers env access:** Import `env` from `cloudflare:workers` at the top of the server function file:

```ts
import { env } from 'cloudflare:workers'
// then use: env.MAILCHIMP_API_KEY, env.MAILCHIMP_AUDIENCE_ID, env.MAILCHIMP_DC
```

This works both locally (via `.dev.vars`) and in production (via `wrangler secret`). The secrets must be declared in `wrangler.jsonc` under `[vars]` for non-secret values, or set via `wrangler secret put` for sensitive ones.

### 3. Create the `SignupModal` component (`src/components/SignupModal.tsx`)

- Props: `{ open: boolean, onClose: () => void, freebie: Freebie }`
- Renders a centered overlay with backdrop blur.
- Contains:
  - Freebie title (so user knows what they're getting)
  - Email input field
  - "Sign me up!" submit button
  - Loading/success/error states
- On submit: calls the server function with `{ email, rewardUrl: freebie.rewardUrl }`.
- On success: shows "Check your inbox!" message, then closes after a delay or on user action.
- Accessible: focus trap, escape to close, `aria-modal`.

## Mailchimp Setup (Manual, not code)

These are manual steps the user must do in the Mailchimp dashboard:

1. **Add merge field** `REWARD_URL` (type: Text) to the audience.
2. **Create an automation** (Classic Automation or Customer Journey):
   - Trigger: "When a subscriber joins the list" (or "Tag is added").
   - Email content includes: `<a href="*|REWARD_URL|*">Download your freebie</a>`.

## Notes

- **Cloudflare Workers env access in TanStack Start:** Resolved — use `import { env } from 'cloudflare:workers'`. Works in both local dev (`.dev.vars`) and production (wrangler secrets). Since the project has `compatibility_date: "2025-09-02"` with `nodejs_compat`, `process.env` would also work, but the `cloudflare:workers` import is the canonical approach.
- **Already-subscribed users:** Mailchimp returns a 400 if the email already exists. We should handle this gracefully (either update the member's merge field, or show a "you're already signed up" message).
- **Rate limiting / spam:** For v1, we won't add rate limiting, but it's worth noting for future hardening.
